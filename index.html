Challenges

Bottelnecks

  • Single Server:

    Apparently when system system scale to its limit, single machine will not be capable to store all the machines. So we need to scale to use multiple instances and server. SOLUTION: The problem can be solved with the help of Load Balancer, which will help to handle the multiple request and prevent the servers from overloading, along with this load balancer will provide us with the static content caching.

  • Concurrency

    There can be multiple users inserting the same URL or editing the same alias at the same time.
    SOLUTION: With a multiple machine this can be solved with the help by creating the Lookup system.

  • Replication/Recovery/Backup

    Data stores can crash for various random reasons.
    SOLUTION: Is having multiple replicas for each database.

  • Response Time

    As the data increases, we need to keep the response time.
    SOLUTION: The solution is to add caching system with the help of system like Redis.
